## ADDED Requirements

### Requirement: README documents E2E test execution
The Frontend-mobile README SHALL include a section titled "Pruebas E2E (Maestro)" that documents how to run the E2E test suite locally.

#### Scenario: README contains installation instructions
- **WHEN** the README is read
- **THEN** it includes steps to install Maestro (`brew install maestro` or `curl -Ls "https://get.maestro.mobile.dev" | bash`)

#### Scenario: README contains execution instructions
- **WHEN** the README is read
- **THEN** it includes the command `maestro test maestro/flows/` for running all flows and `maestro test maestro/flows/study-session.yaml` for a specific flow

#### Scenario: README contains preconditions
- **WHEN** the README is read
- **THEN** it lists preconditions: Android emulator running, backend server running, debug APK installed

### Requirement: README documents CI artifact interpretation
The README SHALL document how to download and interpret artifacts generated by the CI pipeline when E2E tests fail.

#### Scenario: README describes artifact download
- **WHEN** the README is read
- **THEN** it explains how to access CI artifacts (GitHub Actions → failed run → Artifacts section → download)

#### Scenario: README describes artifact contents
- **WHEN** the README is read
- **THEN** it explains that artifacts include screenshots at each step of the failed flow and a video recording in `.mp4` format

#### Scenario: README describes how to re-run failed flows locally
- **WHEN** the README is read
- **THEN** it explains how to use the downloaded Maestro output to replay and debug the failed flow locally
